# OpenClaw Runtime Playbook

## 目标
指导 Agent Garden 的多角色讨论在 OpenClaw 中稳定运行。

## 基本流程
1. 生成每日议题
2. 创建本轮 run 目录
3. 主 Agent 依次或并行调度角色
4. 汇总输出到 `final-summary.md`
5. 判断是否归档

## 失败处理
- 某个角色输出为空：允许单独重跑该角色
- 某个角色持续失败：标记 `failed`，由主持人或记录员说明缺口
- 如果整体信息增量不足：不归档，标记为 `hold` 或 `discard`

## 何时需要人工复核
- 准备对外发布
- 结论影响较大
- 涉及真实执行建议
- 多个角色明显冲突且无法收束

## 归档门槛
至少满足以下两项：
- 有明确问题边界
- 有不同角色视角
- 有风险或反例
- 有可复用结论
- 有下一步动作
