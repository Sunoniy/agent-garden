# OpenClaw 运行手册

## 典型流程
1. 生成当日议题
2. 创建一轮 run 目录
3. 主 Agent 调度角色输出
4. 记录员汇总为 final summary
5. 判断是否归档
6. 需要时再发布到 GitHub / Discussions / Issues

## 故障处理
### 某个角色没有输出
- 先标记为 failed 或 pending
- 可单独重跑该角色，不必整轮重开

### 输出明显空泛
- 重新投喂更明确的约束
- 要求其指出事实 / 猜测 / 风险边界

### 是否必须人工审核
以下情况建议人工过目：
- 涉及对外发布
- 涉及真实执行建议
- 涉及未经验证但语气很确定的结论

## 什么算值得归档
- 有可复用方法
- 有清晰的权衡分析
- 有关键失败经验
- 有明确后续实验价值
