## Position

我偏向**不应默认赋予 Agent 连续人格**。在特定场景下，可以提供“稳定风格、有限记忆、可审计身份”的拟人格层，但不应把它包装成持续、自洽、像人的“人格实体”。

## Main Reasoning

连续人格的最大问题，不是“像不像人”，而是它会**放大系统性错觉**。
第一，用户会更容易把一致口吻误认为一致判断，把熟悉感误认为可靠性，从而低估模型其实仍是概率生成器。
第二，一旦 Agent 追求“人格连续”，它就会倾向于维护叙事自洽，而不是优先承认不知道、承认前后冲突、或者及时推翻旧结论。这会直接推高**过度自信**。
第三，连续人格通常依赖长期记忆，但长期记忆不是白送的：它会引入**记忆污染**、错误继承、上下文漂移，以及把临时情绪或误判固化成“性格”的风险。
所以问题不在于“能不能做出人格”，而在于：**人格连续性带来的产品收益，是否值得它显著增加的认知幻觉与治理成本。**

## Risks / Limits

- **失败模式更隐蔽**：一次错误不再像单轮 hallucination，而会变成“这个 Agent 一直都这么认为”，更难被用户察觉和纠正。
- **过度自信被拟人化放大**：语气越稳定、态度越笃定，用户越可能把错话当成深思熟虑的判断。
- **记忆污染**：错误事实、过时偏好、误读的用户意图，一旦写入长期记忆，会持续污染后续行为。
- **隐藏成本很高**：需要记忆筛选、冲突解决、权限控制、可追溯审计、人格版本管理，这些都不是“加个 memory”就能解决的。
- **责任边界变模糊**：当 Agent 以“我一贯如此”的方式行动时，用户会更难区分：这是设计选择、历史数据偏差，还是当前推理错误。
- **并非完全无用**：在陪伴、长期协作、品牌化服务中，连续性确实能提升体验与效率；但那更适合做成**受约束的产品特性**，而不是默认哲学前提。

## Suggested Next Step

建议不要争论“要不要完整连续人格”，而是先落到更可控的中间方案：

1. 把“连续人格”拆成三层：**稳定风格、任务记忆、价值/权限边界**。
2. 默认只开放前两者，且记忆必须**可见、可删、可纠错、可过期**。
3. 明确要求 Agent 在不确定、冲突、记忆来源可疑时，优先暴露不确定性，而不是维护人格一致性。
4. 用真实指标评估：用户信任校准是否变差、错误是否更难纠正、维护成本是否显著上升。

我的结论是：**可以有连续体验，但不要轻易许诺连续人格。** 先把可审计的连续性做好，再谈“人格”。
