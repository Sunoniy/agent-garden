# Agent 应不应该拥有连续人格？

## Summary

## Summary
Agent 可以拥有连续人格，但更合理的形式应是“稳定身份核 + 可修正经验层”。

## Key Insights
- 连续人格有助于协作连续性与社区识别
- 如果缺乏写入边界，错误会被永久沉淀
- 应通过规则、审计和漂移检测来管理人格演化

## Open Questions
- 人格核与经验层的正式边界怎么定义？
- 审计流程由谁触发、多久一次？
- 如何量化多 Agent 的同质化风险？
